2 switches cisco como si fuera el IDF

2 servidores para produccion

2 pcs de oficina

2 router


Configuracion del switch:

hostname SW-IDF-PLANTA-01
no ip domain look

//Creacion de VLANS
conf t
vlan 10
name OFICINAS
vlan 20
name PRODUCCION

//Asignacion de puertos
interface range f0/1-2
switchport mode access
switchport access vlan 10
exit
interface range f0/10-11
switchport mode access
switchport access vlan 20
exit

interface vlan 99
ip address 10.0.99.1 255.255.255.0
no shut
description Administracion remota
exit

// Contrasenas de acceso
line vty 0 4 
password @D01256SUPA
login
exit
enable secret Sw1tCH2026

Configuracion de direcciones ip segun su VLAN para poder tener una mejor visualizacion

 Oficina (VLAN 10):

PC 01: 192.168.10.1 / Máscara: 255.255.255.0

PC 02: 192.168.10.2 / Máscara: 255.255.255.0

 Planta (VLAN 20):

Nodo 01: 192.168.20.1 / Máscara: 255.255.255.0

Nodo 02: 192.168.20.2 / Máscara: 255.255.255.0

No debe de haber comunicacion entre las vlans pues eso Tambien refuerza la seguridad en el sentido de que si hay un ciberataque o asi que solo sea a un segmento y no se extienda a otras areas.

//Agregar un servidor DHCP
agregar el servidor a la vlan 10
int f0/24
description ServidorDHCP
switchport mode access
switchport access vlan 10
exit

// Etiquetado de cada puerto correspondiente
interface f0/1
description PC-OFICINA-01
interface f0/2
description PC-OFICINA-02
interface f0/10
description Nodo-planta-01
interface f0/11
description Nodo-planta-02
interface f0/24
description ServerDHCP
exit

//Configuracion de seguridad en el puerto del servidor DHCP
interface f0/24
switchport port-security
switchport port-security maximum 1
switchport port-security mac-address sticky
switchport port-security violation shutdown
